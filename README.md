# external-dns-technitium-dns

An [external-dns](https://github.com/kubernetes-sigs/external-dns) webhook provider that manages DNS records on [Technitium DNS Server](https://technitium.com/dns/).

## Overview

`external-dns-technitium-dns` runs as a sidecar alongside external-dns and translates Kubernetes DNS management requests into Technitium DNS Server API calls.

```
Kubernetes resources → external-dns → webhook (this project) → Technitium DNS Server API
```

## Features

- Implements the full [external-dns webhook provider](https://kubernetes-sigs.github.io/external-dns/v0.14.2/tutorials/webhook-provider/) API
- Automatic token refresh on Technitium token expiry
- Record ownership tracking via DNS record comments (`Generated by external-dns-technitium-webhook (resource: ...)`)
- Safety checks: only modifies/deletes **A** and **AAAA** records that were created by this webhook; skips all others with a warning
- Structured JSON logging via `log/slog`
- Minimal container footprint using a `distroless` base image

### Supported record types

| Type  | Notes                                    |
|-------|------------------------------------------|
| A     | IPv4 address                             |
| AAAA  | IPv6 address                             |
| TXT   | Used by external-dns for ownership tracking (txt registry) |

All other record types are ignored.

## Configuration

All settings are provided via environment variables.

| Variable                    | Required | Default   | Description                                          |
|-----------------------------|----------|-----------|------------------------------------------------------|
| `TECHNITIUM_URL`            | ✅       | —         | Base URL of the Technitium DNS server (e.g. `http://localhost:5380`) |
| `TECHNITIUM_USERNAME`       | ✅       | —         | Technitium admin username                            |
| `TECHNITIUM_PASSWORD`       | ✅       | —         | Technitium admin password                            |
| `ZONE`                      | ✅       | —         | DNS zone to manage (e.g. `example.com`)              |
| `DOMAIN_FILTERS`            |          | `$ZONE`   | Semicolon-separated list of domains passed to external-dns during negotiation |
| `LISTEN_ADDRESS`            |          | `0.0.0.0` | Address the webhook server binds to                  |
| `LISTEN_PORT`               |          | `8080`    | Port for the webhook API (also serves `/health` and `/healthz`) |
| `TECHNITIUM_TIMEOUT_SECONDS`|          | `10`      | Per-request HTTP timeout when calling Technitium     |
| `TECHNITIUM_VERIFY_SSL`     |          | `true`    | Set to `false` to disable TLS certificate verification |
| `LOG_LEVEL`                 |          | `INFO`    | Verbosity: `DEBUG`, `INFO`, `WARN`, or `ERROR`       |

## API endpoints

| Method | Path               | Description                                    |
|--------|--------------------|------------------------------------------------|
| GET    | `/`                | Domain filter negotiation                      |
| GET    | `/records`         | Retrieve all DNS records in the managed zone   |
| POST   | `/adjustendpoints` | Endpoint validation pass-through               |
| POST   | `/records`         | Apply DNS changes (returns 204 on success)     |
| GET    | `/healthz`         | Health check                                   |

## Record ownership

Ownership is tracked by external-dns via TXT records (the default `--registry=txt`). When external-dns creates an A/AAAA/CNAME record it also creates a corresponding TXT record containing ownership metadata. It will only ask this webhook to delete records it owns.

## Deployment

### Helm chart (recommended)

The official [external-dns Helm chart](https://github.com/kubernetes-sigs/external-dns/tree/master/charts/external-dns) has first-class webhook sidecar support. Create a `values.yaml` like the following:

```yaml
# values.yaml
sources:
  - ingress
  - service

registry: noop   # or "txt" if you want ownership records

extraArgs:
  - --webhook-provider-url=http://127.0.0.1:8080
  - --no-publish-internal-services
  - --service-type-filter=LoadBalancer

provider:
  name: webhook
  webhook:
    image:
      repository: ghcr.io/xruins/external-dns-technitium-dns
      tag: latest
    env:
      - name: TECHNITIUM_URL
        value: "https://dns.example.com"          # your Technitium server
      - name: ZONE
        value: "example.com"
      - name: DOMAIN_FILTERS
        value: "example.com"                      # semicolon-separated if multiple
      - name: TECHNITIUM_USERNAME
        valueFrom:
          secretKeyRef:
            name: technitium-dns                  # kubectl create secret generic technitium-dns ...
            key: username
      - name: TECHNITIUM_PASSWORD
        valueFrom:
          secretKeyRef:
            name: technitium-dns
            key: password
    livenessProbe:
      httpGet:
        path: /
        port: 8080
    readinessProbe:
      httpGet:
        path: /
        port: 8080
    securityContext:
      allowPrivilegeEscalation: false
      readOnlyRootFilesystem: true
      runAsNonRoot: true
      runAsUser: 65532
      runAsGroup: 65532
```

Install or upgrade:

```bash
# Create the credentials secret first
kubectl create secret generic technitium-dns \
  --from-literal=username=admin \
  --from-literal=password=<your-password>

helm repo add external-dns https://kubernetes-sigs.github.io/external-dns/
helm upgrade --install external-dns external-dns/external-dns \
  --namespace external-dns --create-namespace \
  -f values.yaml
```

> **Note on `DOMAIN_FILTERS`:** the value is a semicolon-separated list (`"example.com;sub.example.com"`).
> The Helm chart example above uses a single domain; adjust as needed.

### Kubernetes sidecar (manual)

```yaml
containers:
  - name: external-dns
    image: registry.k8s.io/external-dns/external-dns:latest
    args:
      - --source=service
      - --provider=webhook
      - --webhook-provider-url=http://127.0.0.1:8080

  - name: external-dns-technitium-dns
    image: ghcr.io/xruins/external-dns-technitium-dns:latest
    env:
      - name: TECHNITIUM_URL
        value: "http://technitium.dns.svc.cluster.local:5380"
      - name: ZONE
        value: "example.com"
      - name: TECHNITIUM_USERNAME
        valueFrom:
          secretKeyRef:
            name: technitium-dns
            key: username
      - name: TECHNITIUM_PASSWORD
        valueFrom:
          secretKeyRef:
            name: technitium-dns
            key: password
    ports:
      - containerPort: 8080
    livenessProbe:
      httpGet:
        path: /
        port: 8080
    readinessProbe:
      httpGet:
        path: /
        port: 8080
    securityContext:
      allowPrivilegeEscalation: false
      readOnlyRootFilesystem: true
      runAsNonRoot: true
      runAsUser: 65532
      runAsGroup: 65532
```

go build ./cmd/main.go
```

```
